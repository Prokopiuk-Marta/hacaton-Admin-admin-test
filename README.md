 КАРИБО Support AI: Hackathon Test Task

Проєкт для автоматичної генерації реалістичних діалогів служби підтримки та їх глибокого автоматизованого аналізу. 

Використовує **Multi-LLM архітектуру** для досягнення максимальної ефективності та об'єктивності:
* **Генерація даних (Google Gemini):** 
* **Строгий логічний аналіз (OpenAI GPT-4o-mini):**

## Ключові особливості
- **Повністю докеризовано:** Запускається в один клік без налаштування локального середовища.
- **Асинхронність:** Використовує паралельні потоки (ThreadPoolExecutor) для блискавичної генерації та аналізу десятків чатів одночасно.
- **Pydantic Validation:** 100% гарантія правильної структури вихідних даних.

## Інструкція із запуску

Для запуску проєкту вам знадобиться лише встановлений [Docker](https://www.docker.com/).

### Крок 1. Клонування репозиторію
Відкрийте термінал, перейдіть у зручну для вас папку та виконайте:
```
git clone [https://github.com/Prokopiuk-Marta/hacaton-Admin-admin-test.git](https://github.com/Prokopiuk-Marta/hacaton-Admin-admin-test.git)
cd hacaton-Admin-admin-test

КРОК 2
Налаштування ключів доступу
Знайдіть у корені проєкту файл .env.example.
Створіть на його основі новий файл з назвою .env.
Вставте туди ваші API-ключі:

КРОК 3
В ТЕРМІНАЛІ:
docker compose up --build

**ДЕ ШУКАТИ РЕЗУЛЬТАТИ?**
Після завершення роботи скриптів у корені вашої папки з'являться два нових файли:
dataset.json — згенеровані сирі діалоги між клієнтами та операторами.
results.json — детальний звіт від аудитора з оцінками
